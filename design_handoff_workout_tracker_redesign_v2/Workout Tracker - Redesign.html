Что не так сейчас

Layout · #1

«Mobile-first» — но на десктоп-ширину

Контейнер max-w-5xl + центрирование. На телефоне всё прижато, на десктопе — пустота по бокам. Карточки не оптимизированы под палец.

Визуальный язык · #2

Стоковый Tailwind: серый фон, синие кнопки

«Спортзал» теряется в общеоффисном UI. Эмодзи 💪 🏋️ 📊 🏠 ⚙️ как декор иконок — выдают «по-быстрому собрано».

Иерархия · #3

Цифры — мелкие, текст — крупный

В зале вы смотрите на телефон одной рукой, мокрой. Главное — вес и повторения. Сейчас они text-sm, как заголовки.

UX логирования · #4

Нет «следующего подхода»

Форма показывает все подходы плоско. Нет акцента на текущий, нет таймера отдыха, нет авто-подсказки из прошлого раза.

Аналитика · #5

Графики есть, но «выводов» нет

Тоннаж и частота показаны, но без сравнения с прошлым периодом, без PR-флагов, без сплита по группам мышц.

Что изменил

Решение · #1

Контент под палец, нижняя навигация

Ширина — телефонная, 5 табов снизу, центральный FAB «+ НОВАЯ» — самое частое действие в один тап с любого экрана.

Решение · #2

Чёрный фон + лаймовый неон

Один яркий цвет #D7FF1A на действиях, PR и тоннаже. Остальное — оттенки чёрного. Узнаваемо «спортзал», не аляписто.

Решение · #3

Цифры — display-кегль

Вес и повторения шрифтом Anton, 22-64px. Подпись «кг» — мелкая. На детали тренировки видно с метра.

Решение · #4

Активный подход + таймер отдыха

На экране «лог» — один подход выделен лаймовой рамкой, кнопка «✓ Подход выполнен», авто-старт таймера 90с. Прошлый результат — подсказка под названием.

Решение · #5

Аналитика — с дельтами и PR

Большие числа сверху, дельта к прошлому периоду, сплит по группам мышц, флаг «PR» прямо в карточке тренировки.

Тренер / Админ

Тренер · #6

Клиенты — с пульсом недели

Список клиентов с прогрессом плана недели (точки) и флагом «● ПРОПУСК», красные клиенты сразу видны. Сверху — KPI всех клиентов разом.

Тренер · #7

Клиент — карта соблюдения

16-недельный grid выполнения плана (как GitHub contributions, но лаймовый). Большая CTA «+ Назначить тренировку».

Тренер · #8

Шаблоны — как карты к применению

Категории табами, на каждой карточке счётчик «применён ×N» — видно, что работает. «Применить» — основная кнопка, «Открыть» — вторичная.

Тренер · #9

Применение — день одним тапом

Семидневный date-picker полосой (в форме сейчас — flatpickr-модалка). Чекбоксы клиентов → внизу sticky CTA «Создать N тренировок».

Админ · #10

Пользователи + Назначения

Таблица из HTML стала мобильной карточкой со счётчиками ролей. Связки тренер↔клиент структурированы по тренерам, а не плоским списком.
